The most likely reason for not reflecting schema changes in the mongodb is that Next.js is still using the old cached Mongoose model.

I export:

export default mongoose.models.User || mongoose.model("User", UserSchema);

When the server starts the first time, mongoose.models.User is created with the old schema. After you edit the schema, Next.js hot reload does not recreate the model, so the new fields are ignored.
